# 示例：外包装设计申请单（自然语言形式）

以下示例展示了用户以自然语言提交申请时，协调智能体应如何提取字段并启动流程。

---

## 示例 1：完整结构化申请

**用户输入**：
```
请帮我设计以下产品的外包装：
- 产品名称：蓝牙音箱 SoundBar Mini
- 产品尺寸：长 240mm，宽 90mm，高 70mm
- 产品重量：0.65kg
- 单箱数量：4 件
- 运输方式：多式联运（陆运+海运）
- 跌落高度要求：1.2m
- 易碎等级：高
- 特殊要求：防潮处理，外箱需印刷品牌 LOGO
- 目标成本：每套不超过 8 元
```

**协调智能体提取的 DesignContext.request_form**：
```json
{
  "product_name": "蓝牙音箱 SoundBar Mini",
  "product_dimensions": { "length_mm": 240, "width_mm": 90, "height_mm": 70 },
  "product_weight_kg": 0.65,
  "quantity_per_box": 4,
  "transport_mode": "multimodal",
  "drop_height_requirement": "1.2m",
  "storage_conditions": "normal",
  "fragility_level": "high",
  "special_requirements": "防潮处理，外箱需印刷品牌 LOGO",
  "target_cost_per_unit_cny": 8.0
}
```

**协调智能体播报**：
```
✓ 申请单解析完成，所有必填字段已获取，启动设计流程...
▶ [Step 1/4] 箱体设计 — 正在检索匹配箱型...
```

---

## 示例 2：信息不完整的申请（触发追问）

**用户输入**：
```
帮我设计一个耳机的外包装，要结实一点，用于出口。
```

**协调智能体识别缺失字段**，播报追问：
```
我已了解您需要为耳机设计出口外包装。为了开始设计，还需要补充几个关键信息：

1. 耳机的尺寸（长×宽×高，mm）是多少？
2. 单件重量大约是多少（kg 或 g）？
3. 每箱计划装几件？
4. 主要运输方式是空运、海运，还是多式联运？
```

---

## 示例 3：包含文件上传的申请

**用户输入**：
```
附件是我们的外包装设计申请单，请按照申请单内容生成设计方案。
```

**协调智能体行为**：
1. 读取上传文件（PDF/Word/Excel）
2. 从文件中提取申请单字段
3. 向用户确认提取结果：
```
我已从您上传的文件中提取到以下信息，请确认：
- 产品名称：[提取值]
- 产品尺寸：[提取值]
...

信息是否正确？确认后我将立即开始设计。
```
