# 箱体匹配算法说明

本文档描述 `scripts/box_matcher.py` 的完整算法逻辑，供调优规则和排查问题时参考。

---

## 算法总流程

```
输入（申请单）
    ↓
1. 生成候选排列矩阵
   ├── 产品数量 N 的因数三元组 (nx, ny, nz)
   └── 超量容差（最多 N×1.25+1 个格位）
    ↓
2. 产品姿态枚举（最多6种排列）
    ↓
3. 计算每种（排列×姿态）所需内箱尺寸
   = (nx×pL + 2×gap, ny×pW + 2×gap, nz×pH + 2×gap)
    ↓
4. 遍历配置表中所有箱型，过滤
   ├── 重量过滤：qty × weight ≤ box.max_weight_kg
   └── 尺寸适配：req_sorted ≤ box_inner_sorted（允许箱体旋转）
    ↓
5. 对每个通过过滤的（箱型，排列，姿态）计算综合评分
    ↓
6. 每个箱型取最佳排列方案
7. 跨箱型按得分排序
    ↓
输出：推荐 + 最多3个备选
```

---

## 关键参数

### 缓冲间隙（clearance）

默认值：**每侧 15mm**（`--clearance 15`）

含义：每个方向上，产品装配轮廓与箱体内壁之间的最小留空，用于放置缓冲材料。

计算方式：每个维度的总留量 = 2 × clearance（两侧各一次）

**协调智能体调整时机**：
- 材料设计完成后，如果实际缓冲厚度超过 15mm，可用 `--clearance <实际值>` 重新运行
- 高脆值产品建议 clearance=20mm，低脆值产品可降至 12mm

## 排列搜索

### 因数三元组

对于 N 件产品，生成满足 a×b×c = N 的所有整数三元组（a≤b≤c）。

示例：
```
N=6 → (1,1,6), (1,2,3), (2,3,1)
N=4 → (1,1,4), (1,2,2)
N=5 → (1,1,5)  ← 质数，只有一种精确排列
```

### 超量容差

当 N 为质数或结果排列方式过少时，允许多预留最多 ⌈N×0.25⌉+1 个格位（空格位不放产品）。

示例：N=5，允许尝试 N=6（(1,2,3)、(1,1,6)）、N=7 等，防止质数情况下排列过于极端。

### 产品旋转

每种产品尺寸 (L, W, H) 最多有 6 种不同摆放方式（全排列）。正立方体情况下自动去重。

## 尺寸适配检查

将 `req_dims` 和 `box_inner` 各自升序排列后逐维对比：

```python
sorted(req_dims)[i] <= sorted(box_inner)[i]  for i in 0,1,2
```

这等效于：只要箱体内尺寸（任意轴向旋转后）能容纳产品排列体积，就认为适配。

**为什么不检查轴向对应关系**：
箱体本身可以任意方向放置（调整哪边朝上/朝外），因此只需确保三个维度的数值匹配，无需固定轴对应。

## 评分公式

```
总分 = 0.35 × 空间利用率得分
     + 0.25 × 重量裕量得分
     + 0.25 × 运输适配得分
     + 0.15 × 尺寸紧凑度得分
```

### 空间利用率得分（35%）

```
space_util = (产品体积 × qty) / 箱体内腔体积

得分：
  util ∈ [55%, 80%]  → 1.0（满分，理想区间）
  util < 55%          → util / 0.55（线性降，箱体过大）
  util > 80%          → (1 - util) / 0.20（线性降，过于紧凑）
  util ≥ 100%         → 0（理论上不应出现）
```

**为什么 55%-80%？**
- 低于 55%：箱体明显偏大，浪费纸箱成本，产品在箱内晃动
- 高于 80%：没有足够空间放缓冲材料，产品防护不足

### 重量裕量得分（25%）

```
weight_util = (qty × 产品重量) / 箱体最大承重

得分：
  util ≤ 80%  → 1.0
  util > 80%  → (1 - util) / 0.20（线性降）
  util > 100% → 0（超重，已被预过滤）
```

保留 ≥20% 的重量安全裕量，防止实际重量（含缓冲材料自重）超出额定值。

### 运输适配得分（25%）

```
完全匹配（transport_mode ∈ box.compatible_transports）→ 1.0
部分兼容（相邻运输方式有重叠）                         → 0.6
完全不匹配                                            → 0.2
```

**multi_modal 语义**：要求最严格，只有 `multimodal` 在箱型支持列表中才得满分；仅支持 `land`/`sea` 的箱型得 0.6 分。

### 尺寸紧凑度得分（15%）

```
excess_ratio = (箱体内腔体积 - 所需最小体积) / 箱体内腔体积

得分 = max(0, 1 - excess_ratio × 2)
```

惩罚体积余量超过 50% 的情况，鼓励选择尺寸更贴合的箱型。

## 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 无任何匹配 | 返回 `status: failed`，含诊断信息和建议操作 |
| N=1（单件装） | 退化为单件排列，clearance 加大到 20mm（可选） |
| N 为质数 | 触发超量容差机制，最多尝试 N+3 |
| 正方体产品 | 6 种旋转退化为 1 种，自动去重 |
| 箱体长宽高相等 | box_inner 所有排列相同，不重复计算 |
| 重量临界（99%承重） | 仍通过过滤，但 weight_score 接近 0，不会成为首选 |

## 调参建议

**场景：推荐结果箱体过大**
→ 降低 `clearance`（如 12mm），或者检查产品是否有更优的摆放姿态

**场景：多式联运始终推荐 BC 楞，但客户想要更轻的 C 楞**
→ 在箱体配置表中为目标 C 楞箱型添加 `"multimodal"` 到 compatible_transports，或修改评分权重

**场景：成本优先，空间利用率权重应更高**
→ 修改 `total_score` 函数中的权重系数（空间利用率 → 0.50，其他相应下调）

**场景：需要增加新箱型**
→ 直接在 `box_config.json` 的 `boxes` 数组中追加记录，无需修改脚本
