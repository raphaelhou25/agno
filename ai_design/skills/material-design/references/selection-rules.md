## §1 多维度评分规则

评分满分 100 分，5 个维度加权求和。评分针对**每个候选主材**独立计算。

### 1.1 强度裕量合理性（30分）

> 核心原则：既要满足强度要求，又要避免过度设计。

```
margin_pct = (actual_ect - target_ect) / target_ect × 100%

得分规则：
  margin_pct < 0%：    0 分（不满足，理论上已被硬约束过滤，不应出现）
  0% ≤ margin < 5%：   25 分（裕量极小，风险偏高）
  5% ≤ margin < 20%：  30 分（最优区间，满分）
  20% ≤ margin < 40%： 22 分（轻度冗余）
  40% ≤ margin < 60%： 15 分（中度过度设计）
  margin ≥ 60%：       8 分（严重过度设计，强烈建议降级）
```

### 1.2 成本匹配度（25分）

```
unit_material_cost = unit_price_rmb_m2 × volume_m2
cost_ratio = unit_material_cost / (budget_rmb_per_unit × 0.60)

得分规则（预算未提供时此项得 12.5 分，即取中间分）：
  cost_ratio ≤ 0.80：  20 分（成本有余，性价比高）
  0.80 < ratio ≤ 1.00：25 分（最优区间，满分）
  1.00 < ratio ≤ 1.25：18 分（轻微超软约束）
  1.25 < ratio ≤ 1.50：10 分（超出软约束较多，已放宽检索）
  ratio > 1.50：       0 分（严重超预算）
```

### 1.3 环保等级（15分）

```
eco_map = { standard: 0, recyclable: 1, biodegradable: 2 }

required_level = eco_map[eco_requirement]（未提供时 = 0）
material_level = eco_map[material.eco_level]

得分规则：
  material_level < required_level：0 分（不满足，理论上已被硬约束过滤）
  material_level == required_level：15 分（满分）
  material_level > required_level：15 分（超出要求，不额外加分，但在 spec_summary 中标注）
```

### 1.4 运输适配性（20分）

```
matched_modes = transport_mode IN material.transport_modes

完全匹配：20 分
不在列表中但材料为全模式（["land","sea","air","multimodal"]）：20 分
材料支持 2~3 种模式但不含当前模式：10 分（用于备选展示）
完全不匹配：0 分（理论上已被硬约束过滤）
```

### 1.5 供应稳定性（10分）

```
得分 = material.supply_stability_score（配置表字段，0~10 → 线性映射到 0~10 分）

stock_status = "limited" 时，得分额外 × 0.7
```

## §2 置信度评估规则

综合置信度（0.0 ~ 1.0）根据以下因素累积扣分后得出。

**基础分**：1.0

| 扣分事件 | 扣分值 | 说明 |
|--------|-------|-----|
| 必填字段有缺失并使用默认值 | 0.08/字段 | 最多扣 0.20 |
| 湿度值使用默认估算 | 0.05 | 常见缺失字段单独列出 |
| 配置表版本已过期 | 0.05 | 可能数据不准确 |
| 主材检索需放宽价格软约束 | 0.06 | 说明预算偏紧 |
| 检索结果仅 1 条（无有效备选） | 0.10 | 选型空间受限 |
| 主备选方案得分差 > 25% | 0.08 | 结果唯一性强，不确定性高 |
| 任意计算参数使用保守估计值 | 0.05/项 | 最多扣 0.15 |
| fragility_level = very_high | 0.10 | 需专业工程师复核 |
| 主材 ECT 裕量 < 5% | 0.12 | 强度边界，风险高 |
| stock_status = "limited" | 0.05 | 供应风险 |

**置信度阈值与行为**：

| 置信度范围 | 行为 |
|----------|-----|
| ≥ 0.85 | 自动确认，通知协调智能体继续，附简要摘要 |
| 0.65 ~ 0.84 | 展示结果摘要 + 不确定点，请求用户确认 |
| < 0.65 | 展示详细结果 + 强调风险 + 提供两个选项（调整输入 / 接受现有方案），同时建议人工专家介入 |

## §3 降级策略

当主选方案成本超预算或强度过度，触发有序降级路径：

### 3.1 主材降级序列

```
BC 双楞（AB 双楞）
    ↓（强度需求降低 或 预算受限）
C 单楞（高规格，175g+）
    ↓
C 单楞（标准规格，150g+）
    ↓
B 单楞（加强型）
    ↓
B 单楞（标准型）
```

**触发条件**：
- 主选方案 ECT 裕量 > 40%，且有较低楞型满足硬约束 → 建议降级
- 主选方案成本 > budget × 0.75 → 自动检索下一档材料

### 3.2 缓冲材料降级序列

```
聚氨酯泡沫
    ↓
EVA 泡棉（高密度）
    ↓
EPE 珍珠棉（加厚）
    ↓
EPE 珍珠棉（标准）
    ↓
蜂窝纸板（仅适用 medium 易碎等级）
```

**触发条件**：
- 主选缓冲材料成本 > budget × 0.20 → 向下检索
- fragility_level = low → 直接跳过缓冲材料

### 3.3 降级结果展示格式

```
当降级路径被触发时，在 alternative_selection 中标注：
{
  "trade_off": "成本↓XX%，BCT裕量↓XX%，适合[具体场景]",
  "downgrade_reason": "主选方案成本超预算XX%，触发降级"
}
```

## §4 人机交互触发条件

以下情况必须暂停流程，向用户展示结果并请求明确决策：

| 触发条件 | 展示内容 | 用户选项 |
|--------|---------|---------|
| 置信度 < 0.85 | 计算参数、主备选方案、不确定点 | 确认主选 / 选备选 / 调整输入参数 |
| 主材 ECT 裕量 < 3% | 强度警告 + 备选方案 | 接受风险 / 选备选 / 提高预算 |
| 预算超出 > 30% | 成本分析 + 降级方案对比 | 接受超预算 / 选降级方案 / 调整预算 |
| fragility_level = very_high | 特殊保护方案 + 专家建议 | 确认方案 / 申请人工专家介入 |
| 无有效备选方案 | 主选方案 + 约束冲突说明 | 放宽某项约束 / 人工介入 |
| 同一阶段迭代 ≥ 3 次 | 所有未解决问题汇总 | 人工专家介入 / 提供额外信息 |

**确认展示模板**：

```
📦 材料设计结果（置信度：{score}）

▌ 关键计算参数
  跌落高度：{drop_height_mm}mm（{formula_basis}）
  目标 BCT：{target_bct_n}N → 湿度修正后：{adjusted_bct_n}N
  目标 ECT：{target_ect_n_per_m / 1000:.1f} kN/m

▌ 主选方案
  [{primary_material_code}] {primary_material_name}
  实际 ECT：{actual_ect} kN/m（裕量 {margin_pct:.1f}%）
  {cushion_line}
  预估材料成本：¥{cost_per_unit}/件

{warnings_block}

▌ 备选方案
  [{alt_material_code}] {alt_material_name}
  权衡：{trade_off}

请确认主选方案，或说明需要调整的参数。
```

## §5 结果警告分级

所有警告写入 `result.warnings` 数组，按严重程度分级：

| 级别 | 标识 | 含义 | 展示方式 |
|-----|-----|-----|---------|
| ERROR | 🔴 | 流程无法继续（如无任何匹配材料） | 必须立即展示，阻断流程 |
| WARNING | 🟡 | 存在风险但可继续（如库存紧张） | 在确认展示中标注 |
| INFO | 🔵 | 提示性信息（如使用了默认值） | 在计算说明中附注 |

```json
{
  "warnings": [
    {
      "level": "WARNING",
      "type": "stock_limited",
      "material_code": "BC-175-115",
      "message": "该材料库存有限，建议尽快确认采购计划"
    },
    {
      "level": "INFO",
      "type": "default_value_used",
      "field": "storage_humidity_pct",
      "default_value": 70,
      "message": "湿度未提供，使用默认值 70%，可能影响 BCT 修正系数准确性"
    }
  ]
}
```
